## Struttura dati salvata

```kotlin
data class RoomRecord(
    val id:            String,   // UUID generato al momento dell'export
    val name:          String,   // nome inserito dall'utente (default: "Stanza")
    val timestamp:     Long,     // epoch ms
    val area:          Double,   // m² (da roomDimensions.area)
    val height:        Double,   // m  (da roomDimensions.height)
    val openingCount:  Int,      // numero aperture (porte + finestre)
    val floorPlanPath: String?,  // path assoluto PNG, null se export fallito
    val glbPath:       String?   // path assoluto GLB, null se export fallito
)
```

## Formato persistente
**JSON array** in `context.filesDir/room_history.json`
- Nessuna dipendenza esterna: usa `org.json.JSONObject/JSONArray` (Android SDK built-in)
- Scrittura sincrona e veloce (file piccolo, <1ms tipico)
- Il file non esiste finché non viene esportata la prima stanza (lazy creation)

### 5. doStopScan() — salvataggio aggiunto
```kotlin
fun doStopScan() {
    val result       = buildResult()          // invariato
    val openingCount = roomModel?.walls?.sumOf { it.openings.size } ?: 0
    session?.pause(); session?.close(); session = null
    mainHandler.post {
        // NUOVO: salva a history prima di finish()
        if (result.getBoolean("success") == true) {
            val dim = result.getJSObject("roomDimensions")
            RoomHistoryManager.save(RoomRecord(...), this)
        }
        // resto invariato
        onScanComplete?.invoke(result)
        val cb = onScanResult
        if (cb != null) { cb(result); onScanResult = null }
        else pendingResult = result
        setResult(RESULT_OK); finish()
    }
}
```

## Rischi di regressione
| Rischio | Probabilità | Note |
|---------|------------|------|

## Checklist manuale di test

### Dialog naming
- [ ] Premere "Esporta" → appare dialog "Nome stanza" con EditText pre-compilato "Stanza"
- [ ] Cambiare il nome (es. "Salotto") → confermare → export procede
- [ ] Premere "Annulla" → il bottone Esporta torna abilitato, nessun crash
- [ ] Lasciare il campo vuoto + confermare → il nome salvato è "Stanza" (default)

### Persistenza
- [ ] Dopo export: chiamare `getRoomHistory()` da JS → la stanza appare nella lista
- [ ] I metadati (nome, area, altezza, openingCount, timestamp) sono corretti
- [ ] `floorPlanPath` e `glbPath` sono path validi e i file esistono
- [ ] Fare una seconda scansione → `getRoomHistory()` restituisce 2 record
- [ ] Chiudere e riaprire l'app → `getRoomHistory()` restituisce ancora i record (persistenza OK)

### Delete
- [ ] `deleteRoom({ id: "..." })` → `deleted: true`, stanza rimossa dalla lista
- [ ] `deleteRoom({ id: "id-inesistente" })` → `deleted: false`, nessun crash

### Nessuna regressione
- [ ] Flow capture (P0/P1/P2/chiudi) invariato
- [ ] Opening placement invariato
- [ ] Export PNG + GLB invariati (stessi file generati)
- [ ] Il risultato ritornato al plugin JS è identico a prima
- [ ] Se la scansione fallisce (< 3 punti): nessun record salvato (success=false)
- [ ] `cancelScan()` non salva nulla (cancelScanAndFinish non chiama doStopScan)
