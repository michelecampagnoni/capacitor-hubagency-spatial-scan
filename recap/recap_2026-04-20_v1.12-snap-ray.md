## Problemi affrontati

### 1. Revert v1.11 — goniometro al cursore rompeva lo snap
v1.11 aveva spostato il centro goniometro all'XZ del cursore per renderlo visibile.
Effetto collaterale: la distanza centro→cursore era ≈ 0 → la condizione `dist > 0.20f`
non scattava mai → nessun snap → cursore sempre fuori asse.

### 2. Goniometro off-screen in TOP mode (post-revert)
Con il goniometro tornato all'ultimo punto confermato, in TOP mode l'utente punta
verso un angolo del soffitto lontano dal punto confermato. Il raggio snap (fisso
`R_OUT + 0.10f`) non raggiungeva il cursore → raggio visualmente spezzato off-screen.

---

## Soluzioni

### Fix 1 — Revert: goniometerCenter = goniometerCenterPt (`ScanningActivity.kt`)

```kotlin
// RIPRISTINATO (era stato modificato in v1.11):
goniometerCenter = if (!openingMode && ...) goniometerCenterPt else null
```

Centro goniometro = ultimo punto floor confermato. Snap math ripristinata al 100%.

### Fix 2 — Snap ray dinamico (`PerimeterRenderer.kt`)

In TOP mode il raggio snap viene esteso dalla distanza centro→cursore + 0.30m:

```kotlin
val snapRayLen = if (topCursorPoint != null) {
    val dx = topCursorPoint[0] - goniometerCenter[0]
    val dz = topCursorPoint[2] - goniometerCenter[2]
    sqrt(dx * dx + dz * dz) + 0.30f
} else 0.50f
```

Il raggio snap attraversa sempre il cursore (anche se distante 2-3m dal centro).
