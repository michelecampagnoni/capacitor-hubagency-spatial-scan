## Problema risolto

### Perdita ambienti intermedi (root cause)
Il sistema precedente aveva `RoomComposerActivity` hardcoded per 2 stanze (roomAId, roomBId).
Con 3 stanze A→B→C, il Composer per B+C non sapeva che B era già posizionata rispetto ad A.
Risultato: il PNG conteneva solo B+C. A spariva. Con N stanze ne restavano sempre solo 2.

## Architettura nuova

### `CompositionGraph.kt` (nuovo file)
Grafo di composizione persistente in `hub_graph.json`.

Struttura dati `RoomWorldTransform`:
- `roomId` — UUID della stanza posizionata
- `parentId` — UUID della stanza parent (per BFS di componente)
- `worldOffsetX`, `worldOffsetZ`, `worldRotRad` — transform ASSOLUTO rispetto alla root della componente
- `confirmedAt` — timestamp

**Invariante:** La root non ha entry nel grafo (transform identità). Ogni altra stanza ha esattamente un entry.

API:
- `addTransform(context, t)` — salva world transform (chiamato da Composer al confirm)
- `getTransform(context, roomId)` — recupera world transform (null = root)
- `getRootId(context, roomId)` — risale al nodo radice traversando i parentId
- `getComponentRoomIds(context, roomId)` — BFS bidirezionale, restituisce tutti i roomId nella stessa componente connessa

### `RoomComposerView.kt` (aggiornato)
Generalizzato da 2 a N+1 poligoni:
- `fixedPolygons: List<Pair<String, List<Pair<Float,Float>>>>` — ambienti già compositi (name, polygon) in world space
- `newRoomPolygon` — nuovo ambiente in spazio locale
- `fixedLinkCenter` — punto di aggancio lato fisso (world space)
- `newRoomLinkCenter` — punto di aggancio lato nuovo (local space)
- `offsetX/offsetZ/rotationRad` — world transform del nuovo ambiente

### `RoomComposerActivity.kt` (riscritto)
Riceve ora: `parentRoomId` + `newRoomId` + `linkKind` + `linkWidth`

**`loadCompositionData()`:**
1. `CompositionGraph.getComponentRoomIds(parentRoomId)` → lista di tutti i roomId già compositi
2. Per ogni roomId: carica JSON, applica world transform → walls e poligono in world space → `fixedExportWalls` + `fixedPolygons`
3. Carica newRoom → `newExportWalls` + `newRoomPolygon`
4. `computeInitialWorldAlignment()` → calcola world transform iniziale per newRoom (allineamento apertura in world space)

**`confirmSave()`:**
1. Salva `RoomWorldTransform(newRoomId, parentRoomId, worldOx, worldOz, worldRot)` nel grafo
2. `generateCombinedFloorPlan()` = `fixedExportWalls` + `transformWalls(newExportWalls, worldOx, worldOz, worldRot)` → FloorPlanExporter
3. Notifica JS con PNG combinato

### `ScanningActivity.kt` (modifica minore)
Intent verso Composer: `roomAId`/`roomBId` → `parentRoomId`/`newRoomId`

## Flusso corretto con 3+ stanze

**Scan A** → save A (nessun entry in hub_graph.json — A è root)

**Scan B** → collega porta di A → `pendingComposerRoomId = A.id`
→ Composer(parentRoomId=A, newRoomId=B)
→ `getComponentRoomIds(A)` = [A] → fixedPolygons = [A]
→ align B rispetto ad A (parentWorldT = null = identity)
→ confirm → `addTransform(B, parentId=A, worldOx, worldOz, worldRot)`
→ PNG(A+B)

**Scan C** → collega porta di B → `pendingComposerRoomId = B.id`
→ Composer(parentRoomId=B, newRoomId=C)
→ `getComponentRoomIds(B)` = [A, B] → fixedPolygons = [A (identity), B (world transform)]
→ fixedExportWalls = walls_A + walls_B_trasformate_in_world_space
→ align C rispetto a B in world space (parentWorldT = B's world transform)
→ confirm → `addTransform(C, parentId=B, worldOx, worldOz, worldRot)`
→ PNG(A+B+C) ← ora A non sparisce più
