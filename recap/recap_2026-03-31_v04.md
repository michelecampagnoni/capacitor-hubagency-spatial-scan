### Round UI/UX — REGRESSIONE INTRODOTTA
Autorizzato lavoro UI/UX. Durante quella sessione è stata introdotta
un'architettura anchor/local-space che ha rotto il motore di capture:

**Cosa è stato aggiunto (ora rimosso):**
- `roomAnchor: Anchor?` — anchor ARCore su hitResult al tap P0
- `confirmedLocal: MutableList<FloatArray>` — vertici in local space
- `capturedHeightM: Float?` — scalare altezza in local context
- `pendingTap: Boolean` — flag tap → consumato in onDrawFrame
- `processConfirm(frame, camera)` — strict hit test per P0/P1/floor
- `undoLocalModel()` — mirror di PerimeterCapture.undo() per local model
- `localToWorldPts()` — conversione local→world via anchor.pose

**Sintomi del crash:**
- P0 impossibile o fortemente instabile (fluttuante)
- P1 non confermabile (bloccato)
- Tutto il resto del flow inutilizzabile

### Revert — v0.4
**Rimossi** tutti i simboli anchor/local-space elencati sopra.

**Ripristinati (baseline world-space):**
```
handlePerimeterTap():
  AWAIT_HEIGHT  → queueEvent { lastReticleWorldFree → addPoint(relH) }
  FLOOR_ONLY    → queueEvent { lastReticleWorld     → addPoint(x,y,z) }

onDrawFrame livePreview:
  lastLivePreview = perimeterCapture.livePreview(normRw[0], normRw[1], normRw[2])

renderer:
  currentWorldPts = frozenPolygon ?: perimeterCapture.getPolygon()

close handler:
  frozenPolygon = perimeterCapture.getPolygon().map { it.copyOf() }
  capturedH     = perimeterCapture.capturedHeight

buildResult:
  finalPolygon = frozenPolygon?.takeIf { it.size == polygon.size } ?: polygon

buildRoomModel:
  poly = frozenPolygon ?: perimeterCapture.getPolygon()

liveDistM:
  from perimeterCapture.getPolygon().lastOrNull()
```
