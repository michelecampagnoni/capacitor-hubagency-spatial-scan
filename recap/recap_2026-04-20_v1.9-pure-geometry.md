## Problema affrontato

ARCore perde periodicamente il tracking visivo (superfici uniformi, luce insufficiente,
movimenti bruschi). Quando recupera, può shiftare il sistema di coordinate world →
i punti confermati mantengono le vecchie coordinate, i nuovi vengono calcolati nel nuovo
sistema → disallineamento visibile della planimetria.

La root cause era `frame.hitTest()`: usava ARCore plane detection + feature point tracking
per calcolare XZ. Quando ARCore era confuso, produceva posizioni sbagliate o inconsistenti.

---

## Soluzione

**Dopo il floor lock: nessun ARCore hit test per il posizionamento XZ.**

Entrambe le modalità usano proiezione geometrica pura:

| Modalità | Funzione | Piano |
|---|---|---|
| FLOOR | `screenToWorldFloorPlane(camera)` | Y = lastFloorY |
| TOP | `screenToWorldTopPlane(camera)` | Y = lastFloorY + wallHeight |

Entrambe: `camera.pose` (orientamento IMU + odometria) → raggio → intersezione con piano noto.

**ARCore continua a fornire:**
- Camera feed (sempre necessario)
- `camera.pose` orientamento (gyro/accelerometro fusion — stabile anche a tracking perso)
- `camera.pose.translation` posizione (odometria — deriva lentamente, ma molto meno della
  plane detection su superfici difficili)

**ARCore non è più usato per:**
- Hit test sui piani rilevati (rimosso)
- Feature point tracking per XZ (rimosso)
- Plane finding (disabilitato dopo floor lock)

## Nuovo: `screenToWorldFloorPlane()`

```kotlin
private fun screenToWorldFloorPlane(camera: Camera): FloatArray? {
    if (!floorAnchor.isLocked) return null
    val camPos = camera.pose.translation
    val camFwd = camera.pose.rotateVector(floatArrayOf(0f, 0f, -1f))
    val dy     = camFwd[1]
    if (dy > -0.01f) return null   // camera non punta verso il pavimento → sticky
    val t = (lastFloorY - camPos[1]) / dy
    if (t < 0.05f || t > 15f) return null
    return floatArrayOf(camPos[0] + t * camFwd[0], lastFloorY, camPos[2] + t * camFwd[2])
}
```

Identica a `screenToWorldTopPlane()` ma per il piano inferiore. Promuove il fallback
geometrico già presente in `screenToWorld()` a percorso primario.

---

## Disable plane finding dopo floor lock

```kotlin
if (floorAnchor.isLocked && !planeFindingDisabled) {
    sess.configure(Config(sess).apply {
        planeFindingMode = Config.PlaneFindingMode.DISABLED
        ...
    })
    planeFindingDisabled = true
}
```

Una volta che lastFloorY è congelato, ARCore non ha più bisogno di rilevare piani.
Disabilitarlo:
- Riduce CPU ARCore (~20-30% processing)
- Elimina una fonte di confusione (piani fantasma su superfici difficili)
- Riduce la probabilità che ARCore perda il tracking

```
Prima del floor lock:
  ARCore hit test → trova superfici → floorAnchor accumula campioni → LOCK

  lastFloorY = congelato
  Plane finding = DISABLED (ARCore non cerca più piani)

  FLOOR mode: camera.pose → raggio → Y=lastFloorY → XZ
  TOP mode:   camera.pose → raggio → Y=lastFloorY+wallH → XZ proiettato a lastFloorY

  Nessun frame.hitTest() in nessuna modalità.
```
