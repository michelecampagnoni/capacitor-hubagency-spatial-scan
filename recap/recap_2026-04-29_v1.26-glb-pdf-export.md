## Novità principali

### 1. GLB 3D Export — `GlbExporter.kt` (riscritta)
File: `capacitor-plugin/android/src/main/java/it/hubagency/spatialscan/GlbExporter.kt`

Genera un file GLTF 2.0 binario (.glb) dalla geometria rilevata:

**Struttura modello:**
- **Primitiva 0 — Muri** (material "Wall", grigio chiaro `[0.88, 0.88, 0.90]`)
  - Ogni parete = uno o più quad `POSITION + NORMAL`
  - Normale perpendicolare al piano del muro `(nx, 0, nz)` per tutti i 4 vertici
  - Intagli per aperture: segmenti solidi + sopra l'apertura (architrave) + sotto (davanzale, solo finestre)
- **Primitiva 1 — Pavimento** (material "Floor", grigio scuro `[0.60, 0.60, 0.62]`)
  - Fan triangulation → sostituita con **ear-clipping** corretto per poligoni non convessi
  - Normale `(0, 1, 0)` per tutti i vertici
  - Shoelace per rilevare winding CW/CCW e garantire il verso corretto

**Layout buffer binario (non-interleaved):**
```
[wall positions] [wall normals] [wall indices] [floor positions] [floor normals] [floor indices]
```
Ogni sezione allineata a 4 byte. 6 accessor + 6 bufferView (o 3 se assenza pavimento).

**API:** `GlbExporter.export(data: RoomExportData, cacheDir: File): String?`

### 2. PDF A3 vettoriale — `FloorPlanExporter.kt`, `SpatialScanPlugin.kt`, `RoomDataLoader.kt`
- `FloorPlanExporter.exportPdf()` genera PDF A3 (1748×2480px a 150dpi) con stessa pipeline vettoriale del PNG
- **Generazione in background** dopo ogni scan (Thread, non blocca UI)
- Path cached in `SharedPreferences("hub_session")["lastPdfPath"]`
- `SpatialScanPlugin.exportPdf()`: controlla cache → se file esiste restituisce subito, altrimenti fallback con `RoomDataLoader.buildExportData(context)`
- `clearSessionData()` rimuove anche `lastPdfPath` per allineamento con il ciclo di sessione

**API TypeScript:** `SpatialScan.exportPdf(): Promise<{ pdfPath: string }>`

### 3. Multi-room GLB — `RoomComposerActivity.kt`
- `generateCombinedFloorPlan()` ora restituisce `Pair<String?, String?>` (pngPath, glbPath)
- Chiama `GlbExporter.export(exportData, cacheDir)` sincronamente dopo il PNG
- Include `glbPath` nell'`onScanComplete` del Composer

### 4. Test app — viewer 3D (`capacitor-welcome.js`, `index.html`)
- Script `<model-viewer>` aggiunto in `index.html` (unpkg CDN, richiede internet al primo caricamento)
- Toggle **"Planimetria 2D" / "Modello 3D"** appare dopo scan se `result.glbPath` è presente
- GLB servito via `Capacitor.convertFileSrc(result.glbPath)` → `capacitor://localhost/...`
- Pinch-to-zoom già presente sul PNG (doppio tap per reset)

**IMPORTANTE per il build:** occorre sempre `npm run build` PRIMA di `npx cap sync android`

### 5. Opening UI fixes — `OpeningRenderer.kt`, `ScanningActivity.kt`

**Bordo inferiore rimosso:**
- `drawWallHighlight()` usava `GL_LINE_LOOP` (4 lati incluso il bordo a terra)
- Cambiato in `GL_LINE_STRIP` da bottom-left → top-left → top-right → bottom-right
- Risultato: 3 lati (sinistro, top, destro), nessuna linea fuxia sul pavimento

**Allineamento fill con perimetro ciano:**
- `baseY` per `OpeningRenderer` ora = `currentWorldPts.firstOrNull()?.get(1)` (stesso riferimento Y di `PerimeterRenderer`)
- Prima usava `frozenFloorY` che poteva driftare rispetto all'Y dei punti catturati, causando un gap visibile tra il fill del muro e il bordo ciano a terra

**Stepper larghezza:**
- Simboli `−` e `+` invece di frecce `←` e `→` nei due cerchi stepper "Larghezza ↔"
