## Problemi affrontati

### 1. Pulsante ALTO/BASSO rimosso
La v1.7 richiedeva un tap manuale per cambiare modalità. Il toggle è stato eliminato.

### 2. Auto-switch da inclinazione camera
Il sistema rileva automaticamente l'inclinazione del telefono usando il vettore forward
della camera ARCore (che internamente fonde giroscopio + accelerometro).

### 3. Bug diagonale (v1.7)
Quando un punto veniva piazzato in TOP mode, il segmento successivo risultava diagonale.

**Root cause:** il goniometro usava `baseReticle = geoFloor`, che in TOP mode (camera verso
l'alto, `dy2 > 0`) ricadeva nel ramo "camera guarda su → XZ forward 3m a lastFloorY".
Questo XZ era completamente diverso da quello calcolato dalla proiezione del piano superiore.
`lastSnappedReticle` conteneva quindi XZ sbagliato → punto confermato in posizione errata →
segmento successivo diagonale rispetto alla planimetria reale.

**Fix:** `baseReticle = if (topModeActive) lastReticleWorld ?: geoFloor else geoFloor`

In TOP mode, `lastReticleWorld` è già aggiornato con l'XZ corretto del piano superiore
(calcolato nel blocco reticle appena prima del goniometro). Il goniometro snap ora opera
sullo stesso XZ del cursore visibile → punto confermato coerente.

## Logica auto-switch

```kotlin
val camFwdAuto = camera.pose.rotateVector(floatArrayOf(0f, 0f, -1f))
// Isteresi: ON sopra 0.20, OFF sotto 0.10
if (camFwdAuto[1] > 0.20f && !reticleTopMode) {
    reticleTopMode = true
    reticleBufIdx = 0; reticleBufFill = 0   // reset buffer al cambio
} else if (camFwdAuto[1] < 0.10f && reticleTopMode) {
    reticleTopMode = false
    reticleBufIdx = 0; reticleBufFill = 0
    lastTopCursorWorld = null
}
```

| Valore camFwdY | Angolo circa | Modalità |
|---|---|---|
| > 0.20 | ~12° verso l'alto | TOP mode ON (cursore arancione) |
| 0.10 – 0.20 | zona isteresi | mantieni modalità corrente |
| < 0.10 | ~6° o meno / verso il basso | FLOOR mode ON |

Il reset del buffer evita blend di campioni floor+top durante la transizione.

## Fix handlePerimeterTap in TOP mode

```kotlin
// PRIMA (v1.7): sempre screenToWorld(forceFloor=true) → null in TOP mode
// DOPO: branch esplicito
if (reticleTopMode) screenToWorldTopPlane(c)
else screenToWorld(f, c, cx, cy, forceFloor = true)
```

In TOP mode, il fresh hit al tap usa la proiezione geometrica pura sul piano superiore
invece dell'hit test ARCore sul pavimento (che ritorna null quando la camera guarda in alto).

## Comportamento v1.8

```
P0 → P1 (altezza) → P2+:

  Telefono inclinato verso il basso / orizzontale:
    → FLOOR mode (comportamento v1.6)
    → cursore arancione assente
    → punta la giunzione muro-pavimento

  Telefono inclinato verso l'alto (>12°):
    → TOP mode attivo automaticamente
    → cursore arancione appare al piano soffitto
    → edge verticale ciano connette top ↔ floor
    → punta il bordo soffitto/angolo superiore
    → tap → punto salvato al pavimento (XZ da top, Y = lastFloorY)

  Tornare a puntare in basso (<6°):
    → ritorna FLOOR mode automaticamente
```
