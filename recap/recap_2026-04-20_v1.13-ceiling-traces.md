## Problemi affrontati

### 1. Cursore arancione sganciato dalla parete live
Il sync post-goniometro sovrascriveva `lastTopCursorWorld` con l'XZ snappato
(`lastSnappedReticle`). Il cursore arancione si spostava sulla posizione snap,
non sul centro schermo (mirino). I due non coincidevano → cursore "flottante"
non agganciato alla parete.

**Principio corretto:** il cursore arancione = proiezione del centro schermo
(mirino) sul piano superiore. Non è la posizione snappata — quella è il ghost
corner a terra e il `livePreview`.

### 2. Assenza di tracce al soffitto
In TOP mode l'utente guarda verso l'alto e non vedeva né i segmenti confermati
né la linea live al livello soffitto. Solo il pavimento (invisibile da sopra)
aveva le tracce ciano/fucsia. Il feedback visivo era insufficiente.

### 3. Edge verticale non ancorato al mirino
La sezione 5c disegnava l'edge verticale all'XZ di `livePoint` (ghost corner).
In TOP mode dovrebbe essere all'XZ del cursore (mirino), così l'edge è sotto il
punto esatto dove l'utente sta puntando.

---

## Soluzioni

### Fix 1 — Cursore segue il mirino, non lo snap (`ScanningActivity.kt`)

Rimossa la riga che sovrascriveva `lastTopCursorWorld` nel sync post-goniometro:

```kotlin
// PRIMA (v1.12 — errato):
val sr = lastSnappedReticle
if (gonioCenterPt != null && sr != null) lastLivePreview = sr
if (topModeActive && sr != null)
    lastTopCursorWorld = floatArrayOf(sr[0], lastFloorY + wallHeightPreview, sr[2])

// DOPO (v1.13 — corretto):
val sr = lastSnappedReticle
if (gonioCenterPt != null && sr != null) lastLivePreview = sr
// lastTopCursorWorld NON modificato qui — rimane al mirino (normRw dal branch reticle)
```

`lastTopCursorWorld` viene impostato **solo** nel branch reticle con `normRw`
(media 8 frame del centro schermo proiettato sul piano superiore).

### Fix 2 — Edge verticale al cursore in TOP mode (`PerimeterRenderer.kt` sezione 5c)

```kotlin
// XZ dell'edge: cursore (mirino) in TOP mode, ghost corner in FLOOR mode
val vcx = if (topCursorPoint != null) topCursorPoint[0] else livePoint[0]
val vcz = if (topCursorPoint != null) topCursorPoint[2] else livePoint[2]
drawPrimitive(GL_LINES, floatArrayOf(vcx, drawY, vcz, vcx, topY, vcz), 2)
```

L'edge verticale è esattamente sotto il cursore arancione in ogni modalità.

### Fix 3 — Sezione 5e: tracce al soffitto (`PerimeterRenderer.kt`)

Nuova sezione dopo 5d, attiva solo quando `topCursorPoint != null` (TOP mode):

```kotlin
// Segmenti confermati al soffitto (ciano, GL_LINE_STRIP)
// Segmento live al soffitto (fucsia, dall'ultimo punto al cursore/mirino)
// Dot vertex al soffitto
```

Le tracce al soffitto sono lo specchio esatto di quelle al pavimento,
centrate sui punti confermati con l'estremità aperta al cursore (mirino).
